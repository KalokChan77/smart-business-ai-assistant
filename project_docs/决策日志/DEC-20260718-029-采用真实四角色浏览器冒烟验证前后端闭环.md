# DEC-20260718-029：采用真实四角色浏览器冒烟验证前后端闭环

- 日期：2026-07-18
- 状态：已接受
- 参与角色：学生开发者、AI 编程助手
- 关联任务：Vue3 与 FastAPI 联调、Dify 配置验证、四角色业务闭环验收
- 关联实习日记：`project_docs/实习日记/2026-07-18-实习日记.md`
- 关联决策：`DEC-20260717-027`、`DEC-20260718-028`

## 一、背景

后端已有较完整的自动化测试，前端也有单元/组件测试。但对教学演示项目而言，最终验收仍要证明：真实浏览器中的四类角色可以完成登录、分流、AI 对话、知识问答、客服确认和统计查看。接口测试通过并不自动等于页面交互、SSE 完成态、百分比展示和文案承诺都正确。

## 二、需要解决的问题

1. 如何在不污染正式演示数据的前提下做真实联调；
2. 如何验证 DeepSeek 流式对话、Dify 知识问答、客服人工确认和统计页面；
3. 发现展示或交互缺陷后如何快速定位是前端契约问题还是上游服务问题；
4. 测试结束后如何清理临时账号、业务数据和浏览器会话，并保持敏感信息脱敏。

## 三、可选方案

### 方案 A：只依赖自动化测试和接口冒烟

- 优点：执行快，结果稳定。
- 缺点：无法覆盖真实 SSE 完成态、页面按钮出现时机、角色默认落地页和响应式布局。

### 方案 B：使用临时租户与四角色账号做真实浏览器冒烟

- 优点：覆盖完整交互；可验证权限分流；测试数据可按租户隔离并删除。
- 缺点：需要准备临时账号，并严格清理凭据。

### 方案 C：直接使用长期演示账号反复测试

- 优点：准备成本低。
- 缺点：容易污染演示数据，密码和状态更难管理。

## 四、最终决策

采用方案 B，并固定以下验收规则：

1. 创建独立临时租户和四角色账号，凭据仅写入本机临时文件并设置严格权限；
2. 浏览器只访问 Vue3 与 FastAPI 代理路径，不在前端暴露模型 Key、Dify Key 或 Dataset ID；
3. 四角色默认落地页必须分别验证：
   - 企业用户：`/app/chat`
   - 客服：`/service/tickets`
   - 管理员：`/admin/overview`
   - 决策者：`/decision/overview`
4. 业务闭环至少覆盖：
   - DeepSeek 流式回答与 `message_end` 后的反馈入口；
   - Dify 知识问答召回与回答；
   - 用户创建工单、客服分类、生成建议、人工确认；
   - 用户查看已确认最终回复；
   - 管理员与决策者统计页面；
5. 百分比契约统一按后端 0–100 展示，前端不得再次乘 100；
6. `ReplySuggestion.status` 仅使用 `draft | confirmed`；
7. 测试结束后必须：退出登录、删除临时租户业务数据与用户角色、删除临时凭据文件、关闭浏览器标签。

## 五、决策理由

1. **真实交互优先**：SSE 完成态、按钮出现时机和布局溢出只有浏览器才能发现；
2. **数据隔离**：临时租户避免污染正式演示；
3. **责任边界清晰**：AI 只生成建议，客服必须人工确认最终回复；
4. **安全与脱敏**：文档只记录“已配置/已验证”和删除计数，不记录密钥、密码、令牌或 Dataset ID。

## 六、影响与风险

- 正面影响：形成可重复的前后端闭环验收方法，并能直接产出实习日记素材；
- 潜在风险：上游 Dify Nginx 若缓存旧上游地址，会表现为 502，容易被误判为 Key 错误；
- 回退方案：若浏览器环境不可用，可退回接口冒烟，但不得把接口成功直接宣称成“四角色演示完成”。

## 七、验证证据

### 配置与上游

- DeepSeek、百炼、Dify Chat/Workflow/Dataset 配置均已加载；
- 初次 Dify 三类 API 返回 502，根因是 Nginx 缓存了 API 容器旧地址；重启 Nginx 后三类 `/v1/parameters` 或 Dataset 查询均恢复 HTTP 200；
- 文档不记录真实 Key、Dataset ID 或密码。

### 浏览器四角色闭环

- 企业用户：DeepSeek 流式回答完整，`message_end` 后出现反馈按钮；Dify 知识问答结果为 `answered` 并召回知识片段；创建模拟退款工单成功；历史会话可查看完整 AI 回答；确认后可查看客服最终回复；
- 客服：可查看工单、执行 AI 分类、生成回复建议，并人工编辑确认；确认后工单状态变为已解决；
- 管理员：概览异步加载后显示租户用户与统计；用户与角色、知识文档、数据统计页面正常；
- 决策者：经营总览、咨询趋势、问题分类、满意度、AI 质量五页正常，百分比按 0–100 契约展示。

### 缺陷修复

- 分类置信度错误显示为 `8000.0%`，原因是后端已是 0–100，前端重复乘 100；修复后显示 `80.0%`，并新增 `formatPercent` 回归测试；
- “我的工单”文案承诺可查看最终回复，但原先无详情入口；补齐“查看详情”并复用 `GET /customer-service/tickets/{id}` 后，用户可看到确认回复与确认时间。

### 自动化回归

- 前端：typecheck / 21 tests / build / audit 通过；
- 后端：288 tests、compileall、pip check、alembic check 通过。

### 清理

- 临时租户删除结果：`ai_runs=1`、`conversations=1`、`customer_tickets=1`、`reply_suggestions=1`、`roles=4`、`users=4`，剩余用户数 0；
- 临时凭据文件已删除，浏览器测试标签已关闭。
