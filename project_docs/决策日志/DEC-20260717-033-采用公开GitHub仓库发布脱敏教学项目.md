# DEC-20260717-033：采用公开 GitHub 仓库发布脱敏教学项目

- 日期：2026-07-17
- 状态：已接受
- 参与角色：学生开发者、AI 编程助手
- 关联任务：项目开源发布与朋友拉取部署
- 关联实习日记：`project_docs/实习日记/2026-07-17-实习日记.md`
- 关联决策：`DEC-20260717-002`、`DEC-20260717-006`、`DEC-20260717-017`、`DEC-20260717-022`

## 一、背景

项目已完成本地演示和主要功能验收，需要通过 GitHub 公开仓库提供给同学或朋友拉取、学习和部署。公开内容既要保留可运行代码、模拟数据和教学过程材料，也必须避免真实 API Key、密码、Token、Cookie、Dify Dataset ID、本地业务数据和教师提供的二进制资料进入 Git 历史。

## 二、需要解决的问题

1. 哪些代码、文档和模拟数据适合公开，哪些本地文件必须排除；
2. 是否把完整 Dify 官方源码一并提交，还是只发布项目所需覆盖层；
3. 如何证明公开快照不依赖开发机的私有 `.env`，并且没有可识别密钥；
4. 如何让朋友在新环境中得到可复现的安装、测试和部署入口；
5. 采用何种开源许可证，并如何持续检查后续提交。

## 三、可选方案

### 方案 A：只发送私有压缩包

- 优点：发布操作简单，公开风险较低；
- 缺点：不便于协作、更新和版本追踪，也无法使用 GitHub Actions 持续验证。

### 方案 B：把本机项目目录和完整 Dify 源码全部提交

- 优点：表面上看起来内容最完整；
- 缺点：容易把私有配置、生成数据和嵌套仓库带入历史，重复分发大体积第三方源码，也不利于审计。

### 方案 C：公开脱敏后的业务项目，并单独发布 Dify 覆盖层

- 优点：保留完整业务代码、模拟知识库、模拟验收数据、API 文档、决策日志和实习日记；仓库体积小；第三方边界清楚；可以通过示例环境文件和 CI 验证可复现性；
- 缺点：部署者需要按文档另行克隆指定版本的 Dify，并自行配置服务密钥。

## 四、最终决策

采用方案 C：

1. 创建公开 GitHub 仓库 `18and02/smart-business-ai-assistant`，默认分支为 `main`；
2. 使用 MIT License，允许教学、学习和二次开发，同时保留许可证声明；
3. 公开 `backend/`、`frontend/`、`deploy/`、`scripts/`、技术方案、API 契约、模拟知识库、模拟验收 CSV、决策日志和实习日记；
4. 不公开根 `.env`、`project_docs/key env/`、Dify 私有 `.env`、完整 `dify-self-host/`、本地虚拟环境、依赖目录、构建产物、数据库与上传数据、教师提供的 DOCX/PDF、临时文件和 IDE 配置；
5. Dify 1.15.0 由部署者从官方仓库单独克隆，本项目仅发布 Jieba 镜像、共享存储挂载和重建脚本组成的覆盖层；
6. 提供根 `.env.example` 和前端示例环境文件，所有敏感字段保持为空或使用明确占位值；
7. GitHub Actions 对后端单元测试、Python 编译与依赖检查、前端类型检查/测试/构建、Shell 语法和业务 Compose 配置进行持续验证；
8. 首次推送前只扫描 Git 暂存快照，不扫描或读取被忽略的真实配置文件。

## 五、决策理由

公开仓库最适合朋友直接克隆和后续协作，但 Git 历史一旦包含密钥，即使删除文件也不能视为安全，因此必须在第一次提交前完成文件范围审计和密钥扫描。仅发布项目覆盖层可以避免重复维护 574 MB 的 Dify 官方源码，同时把本项目对 Dify 的必要改动清楚地记录下来。MIT License 规则简洁，适合本科教学项目。模拟数据和过程文档能够帮助读者理解业务场景与技术决策，因此在确认不包含个人信息后继续保留。

## 六、影响与风险

- 正面影响：朋友可以通过标准 Git 流程拉取；代码、部署说明和教学证据集中管理；每次推送可以自动回归；
- 潜在风险：公开仓库无法替代生产安全审计；部署者仍需自行准备 Dify、数据库、Redis 和第三方 AI 服务额度；历史日记较长，会增加仓库文本体积；
- 回退方案：如发布前发现问题，停止创建远端并修正暂存快照；如发布后发现敏感信息，应立即撤销对应凭据、清理 Git 历史并重新审计，不能只删除最新版本文件。

## 七、验证证据

- `.gitignore` 和 `git check-ignore` 已确认真实 `.env`、Key CSV、Dify 源码及私有配置、虚拟环境、前端依赖与构建目录、教师 DOCX/PDF、临时目录和根 `AGENTS.md` 不进入 Git 索引；
- 暂存快照未包含二进制文件，最大公开文件为前端锁文件，仓库文本规模约 1.46 MB；
- Gitleaks 对暂存导出快照扫描结果为 `no leaks found`，补充模式扫描未发现长格式模型 Key、GitHub Token、Bearer Token、私钥或长格式 Dify Key；
- 示例环境文件中的模型和 Dify Key 字段均为空，密码与 JWT 只使用明确占位值；
- 在不含私有 `.env` 的全新快照中，Python 3.12 依赖安装成功，后端 `284 passed, 7 deselected`，`compileall` 和 `pip check` 通过；
- 同一快照中，前端 TypeScript 类型检查通过，8 个测试文件共 23 项测试通过，生产构建成功，`npm ci` 未发现已知漏洞；
- 业务 Compose 和 Dify 1.15.0 覆盖层 Compose 均可使用示例配置成功解析，全部 Shell 脚本通过语法检查；
- GitHub 远端公开可见性、首次推送和 Actions 结果在远端创建后补充确认。
