# DEC-20260717-032：将 Economy 关键词检索零分规范为未知相关度

- 日期：2026-07-17
- 状态：已接受
- 参与角色：学生开发者、AI 编程助手
- 关联任务：知识问答引用相关度展示修复
- 关联实习日记：`project_docs/实习日记/2026-07-17-实习日记.md`
- 关联决策：`DEC-20260717-018`、`DEC-20260717-019`

## 一、背景

答辩演示时发现知识问答的每个引用都显示“相似度 0.0000”。真实调用 Dify Dataset 检索后确认，在当前 `economy + keyword_search` 配置下，5 条有效命中记录的 `score` 均为 `0`。该值不是语义相似度，也不能表达命中质量；原样展示会让用户误以为检索完全不相关。

## 二、需要解决的问题

1. `score=0` 应解释为真实零相似度，还是上游未提供可用相关度；
2. 应只在前端隐藏，还是在服务端统一契约；
3. 如何保留未来 Dify 返回正相关度时的展示能力。

## 三、可选方案

### 方案 A：继续展示 0.0000

- 优点：完全原样透传；
- 缺点：数值没有信息量且误导用户。

### 方案 B：仅前端隐藏零分

- 优点：改动小；
- 缺点：API 和客服建议持久化仍保留误导值，其他客户端还会遇到同一问题。

### 方案 C：适配器把关键词检索零分规范为 `null`，前端只展示正相关度

- 优点：第三方哨兵值在边界统一翻译；API 语义稳定；未来正分仍可展示；
- 缺点：不再逐字保留上游的零值。

## 四、最终决策

采用方案 C：

1. `DifyDatasetClient` 在固定 economy 关键词检索响应中，将 `score == 0` 转换为 `None`；
2. `/knowledge/query` 对应输出为 `citations[].score = null`；
3. Vue3 只在分数大于 0 时展示，并将标签从“相似度”改为更准确的“相关度”；
4. 空召回仍以 `records=[]` 判断，不能以分数判断；
5. Dify 若返回正分，继续按 4 位小数展示。

## 五、决策理由

第三方协议的无信息量哨兵值应在适配器边界转换，而不是泄漏到领域和界面。`null` 表达“未提供可用分数”，比 `0` 表达“明确完全不相关”更准确，同时保持知识记录、客服建议和前端的统一语义。

## 六、影响与风险

- 正面影响：不再出现整页 0.0000；相关度展示更符合真实检索方式；
- 潜在风险：若未来检索策略改变并认为零分有业务意义，需要重新评估规范化规则；
- 回退方案：移除适配器零分转换即可恢复原始值，API 字段类型无需变化。

## 七、验证证据

- 修复前真实 Dify 查询返回 5 条记录，分数为 `0,0,0,0,0`；
- 修复后使用同一问题真实检索，仍返回 5 条记录，但公开分数均为 `null`，不再向界面传播无信息量零值；
- 新增后端测试验证零分转为 `None`，原有正分 `0.25` 保持不变；
- 新增前端组件测试验证 `null` 不显示相关度、正分显示“相关度 0.8750”；
- 后端全量 291 项测试通过，`compileall`、`pip check`、`alembic check` 通过；
- 前端 8 个测试文件、23 项测试通过，TypeScript 类型检查、生产构建和生产依赖审计通过。
